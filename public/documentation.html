myFlix API Documentation

Welcome to the myFlix API documentation. This API provides access to a database of movies, directors, genres, and users with full CRUD functionality.

Movies Genres Directors Actors/Actresses Users Data Models Database

Movie Endpoints

GET /movies - Get all movies

Returns a list of all movies in the database.

Postman Test Result:

Status: 200 OK

Response: Returns an array of movie objects

Example Request:

fetch('http://localhost:8080/movies')
  .then(response => response.json())
  .then(data => console.log(data));

User Endpoints

GET /users - Get all users

Returns a list of all registered users (passwords are hashed).

Postman Test Result:

Status: 200 OK

Response: Returns array of user objects

Creates a new user account. Password is automatically hashed before storage.

Request Body:

{

  "Password": "securepassword",

  "FavoriteMovies": []

Data Models

Movie Model (models.js)

let movieSchema = mongoose.Schema({
  Title: { type: String, required: true },
  Description: { type: String, required: true },
  Genre: {
    Name: String,
    Description: String,
  },
  Director: {
    Name: String,
    Bio: String,
    Birth: Date,
    Death: Date,
  },
  Actors: [String],
  Actresses: [String],
  ReleaseYear: Number,
  IMDbRating: Number,
  ImagePath: String,
  Featured: Boolean,
});

User Model (models.js)

let userSchema = mongoose.Schema({
  Username: { type: String, required: true },
  Password: { type: String, required: true },
  Email: { type: String, required: true },
  Birthday: Date,
  FavoriteMovies: [{ type: mongoose.Schema.Types.ObjectId, ref: "Movie" }],
});

// Password hashing methods
userSchema.statics.hashPassword = function (password) {
  return bcrypt.hashSync(password, 10);
};
userSchema.methods.validatePassword = function (password) {
  return bcrypt.compareSync(password, this.Password);
};

Authentication

The API currently uses basic authentication. Passwords are securely hashed using bcrypt before storage with a salt factor of 10.

Error Handling

The API returns appropriate HTTP status codes:

  • 200 - OK (successful request)
  • 201 - Created (successful creation)
  • 400 - Bad Request (invalid input or username exists)
  • 404 - Not Found (resource doesn't exist)
  • 500 - Internal Server Error (server-side problem)

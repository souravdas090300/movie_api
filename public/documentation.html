myFlix API Documentation

Welcome to the myFlix API documentation. This API provides access to a database of movies, directors, genres, and actors.

User Endpoints

POST /users - Register a new user

Creates a new user account.

Request Body:

Field Type Description
Username string Unique username for the user
Password string User's password
Email string User's email address
Birthday Date User's birth date (format: DD/MM/YYYY)

Example Request:

post('http://localhost:8080/users', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    "Username": "francescom",
    "Password": "muscat@123",
    "Email": "francesca.muscat@example.com",
    "Birthday": "15/04/2662"
  })
})
.then(response => response.json())
.then(data => console.log(data));

Success Response (201 Created)

{
  "Username": "francescam",
  "Email": "francesca.muscat@example.com",
  "FavoriteMovies": [],
  "_id": "685418076481525805272401"
}
PUT /users/:username - Update user information

Updates an existing user's information.

Parameters:

Parameter Type Description
username string Username of the user to update

Request Body:

Any of the user fields to be updated (Username, Password, Email, Birthday)

Example Request:

fetch('http://localhost:8080/users/francescam', {
  method: 'PUT',
  headers: {
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    "Username": "francescam",
    "Password": "muscat@123",
    "Email": "francesca.muscat@example.com"
  })
})
.then(response => response.json())
.then(data => console.log(data));

Success Response (200 OK)

{
  "_id": "6854180784815258052724c1",
  "Username": "francescam",
  "Password": "muscat@123",
  "Email": "francesca.muscat@example.com",
  "FavoriteMovies": [
    "684026918dda38a4fe6c4bd8",
    "684826918dda38a4fe6c4bda",
    "684826918dda38a4fe6c4bdc",
    "684826918dda38a4fe6c4bda"
  ]
}
POST /users/:username/movies/:movieId - Add movie to favorites

Adds a movie to a user's list of favorite movies.

Parameters:

Parameter Type Description
username string Username of the user
movieId string ID of the movie to add to favorites

Success Response (200 OK)

Returns updated user document with the new movie in FavoriteMovies array

Error Handling

The API returns appropriate HTTP status codes:

  • 200 - OK (successful request)
  • 400 - Bad Request (invalid input)
  • 404 - Not Found (resource doesn't exist)
  • 500 - Internal Server Error (server-side problem)

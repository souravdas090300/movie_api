myFlix API Documentation

Welcome to the myFlix API documentation. This API provides access to a database of movies, directors, genres, and actors.

Movies Genres Directors Actors/Actresses Data Models Database

Movie Endpoints

GET /movies - Get all movies

Returns a list of all movies in the database.

Example Request:

fetch('https://your-api-url.com/movies')
  .then(response => response.json())
  .then(data => console.log(data));

Success Response (200 OK)

GET /movies/:title - Get movie by title

Returns details about a specific movie by title.

Parameters:

Parameter Type Description
title string Title of the movie to retrieve (case sensitive)

Example Request:

fetch('https://your-api-url.com/movies/Titanic')

Genre Endpoints

GET /genres - Get all genres

Returns a list of all genres in the database with their descriptions.

Success Response (200 OK)

[
  {
    "_id": "Drama",
    "description": "Emotionally intense stories exploring human conflicts..."
  },
  {
    "_id": "Science Fiction",
    "description": "Futuristic worlds, advanced tech, space adventures..."
  }
]
GET /genres/:name - Get movies by genre

Returns all movies belonging to a specific genre.

Example Request:

fetch('https://your-api-url.com/genres/Science%20Fiction')
  .then(response => response.json())
  .then(data => console.log(data));

Success Response (200 OK)

Returns array of movies in the specified genre

Data Models

Movie Model

let movieSchema = mongoose.Schema({
  Title: { type: String, required: true },
  Description: { type: String, required: true },
  Genre: {
    Name: String,
    Description: String,
  },
  Director: {
    Name: String,
    Bio: String,
    Birth: Date,
    Death: Date,
  },
  Actors: [String],
  Actresses: [String],
  ReleaseYear: Number,
  IMDbRating: Number,
  ImagePath: String,
  Featured: Boolean,
});

Database Information

MongoDB Connection

mongoose.connect("mongodb://localhost:27017/myFlixDB", {
  useNewUrlParser: true,
  useUnifiedTopology: true,
});

Error Handling

The API returns appropriate HTTP status codes:

  • 200 - OK (successful request)
  • 400 - Bad Request (invalid input)
  • 404 - Not Found (resource doesn't exist)
  • 500 - Internal Server Error (server-side problem)
